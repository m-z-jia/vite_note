# 组件间调用

![组件使用示例](/guide/组件使用.png)

这是 kebab-case（短横线隔开式） 写法。

在 Vue 中，组件命名的调用规则如下：

- 自动转换 ：当你在代码中以 PascalCase（大驼峰式） 导入一个组件（如`VcToolBar`）时，Vue 的模板编译器允许你在 `<template>` 中使用两种写法：

  ```vue
  <script setup>
  // 导入组件
  // import VcToolBar from './VcToolBar.vue' // 示例组件路径
  </script>
  ```
  
  ```vue
  <VcToolBar />
  <!-- 大驼峰，通常在 .vue 文件内部推荐使用 -->
  <vc-tool-bar />
  <!-- 短横线，符合 HTML 标准标签的视觉风格 -->
  ```

- 为什么这么写？ ：这两种写法在功能上完全等价。但在很多成熟的组件库（如 Element Plus 的 `<el-button>`）中，为了遵循 W3C 的自定义元素规范，习惯上会使用短横线写法。

底层原理：

1. 数据流 (Props) ： `tools` 数据通过属性绑定传给 `VcToolBar` 内部，决定了工具栏显示哪些图标和文字。

   ```vue
   <VcToolBar :tools="toolList" />
   ```

2. 事件流 (Events) ：当你在界面上点击工具栏的某个按钮时， `VcToolBar` 内部会通过 `emit` 发出 `action` 或 `active` 事件，父组件通过 `@` 符号捕获这些事件并执行相应的业务逻辑。

   ```vue
   <VcToolBar @action="handleAction" @active="handleActive" />
   ```
