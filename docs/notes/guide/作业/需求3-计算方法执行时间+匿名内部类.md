# 计算方法执行时间 + 匿名内部类

## 第一部分：计算一个方法执行了多少秒

### 一、基本方式

#### 1. System.currentTimeMillis()（最常用）
```java
long start = System.currentTimeMillis();
doSomething();
long end = System.currentTimeMillis();
System.out.println("耗时: " + (end - start) + "ms");
```
- 返回毫秒级时间戳
- 适合**秒级/毫秒级**测量
- 受系统时间调整影响

#### 2. System.nanoTime()（更精确）
```java
long start = System.nanoTime();
doSomething();
long end = System.nanoTime();
System.out.println("耗时: " + (end - start) / 1_000_000.0 + "ms");
```
- 返回纳秒级计数器
- 适合**微秒/纳秒级**测量
- 不受系统时间调整影响

### 二、封装通用计时工具

```java
// 无返回值版本
public static long measureTime(Runnable task) {
    long start = System.currentTimeMillis();
    task.run();
    return System.currentTimeMillis() - start;
}

// 有返回值版本（使用 Supplier）
public static <T> T measureTimeAndReturn(Supplier<T> task) {
    long start = System.currentTimeMillis();
    T result = task.get();
    System.out.println("耗时: " + (System.currentTimeMillis() - start) + "ms");
    return result;
}
```

### 三、使用方式

```java
// 匿名内部类方式
long time = measureTime(new Runnable() {
    @Override
    public void run() {
        doHeavyWork();
    }
});

// Lambda 方式（更简洁）
long time = measureTime(() -> doHeavyWork());

// 有返回值
String result = measureTimeAndReturn(() -> fetchDataFromDB());
```

## 第二部分：匿名内部类的使用场景

### 什么是匿名内部类？

匿名内部类是**没有名字的内部类**，声明和实例化在一条语句中完成。本质是**一个实现了某个接口或继承了某个类的无名子类对象**。

### 四大经典场景

#### 场景1：实现接口（最经典）

```java
Runnable task = new Runnable() {  // 实际上是创建了一个 Runnable 的匿名实现类
    @Override
    public void run() {
        System.out.println("执行任务");
    }
};
new Thread(task).start();
```

#### 场景2：Comparator 排序

```java
List<String> list = Arrays.asList("C", "A", "B");
Collections.sort(list, new Comparator<String>() {
    @Override
    public int compare(String a, String b) {
        return a.compareTo(b);
    }
});
```

#### 场景3：回调/事件监听

```java
button.setOnClickListener(new OnClickListener() {
    @Override
    public void onClick() {
        System.out.println("按钮被点击了！");
    }
});
```

#### 场景4：继承抽象类

```java
Animal dog = new Animal() {
    @Override
    void speak() {
        System.out.println("汪汪！");
    }
};
```

### 匿名内部类 vs Lambda 表达式

| | 匿名内部类 | Lambda 表达式 |
|------|----------|-------------|
| 适用 | 接口、抽象类 | 仅函数式接口（单抽象方法接口） |
| 代码量 | 较多 | 简洁 |
| this 指向 | 匿名类对象自身 | 外部类对象 |
| 底层原理 | 生成单独的 .class 文件 | invokedynamic 指令 |

**现代 Java（8+）推荐**：如果接口是函数式接口，优先使用 Lambda；如果需要继承类或实现多方法接口，仍然使用匿名内部类。
