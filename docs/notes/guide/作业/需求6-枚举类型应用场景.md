# 枚举类型应用场景

## 枚举的本质

Java 枚举（enum）本质上是**一种特殊的类**，它继承自 `java.lang.Enum`，可以有**字段、构造方法、普通方法和抽象方法**。与普通类的区别是：枚举的实例是**预定义的有限个常量**。

## 场景1：状态/类型定义（最常用！）

### 问题
用 `int` 或 `String` 常量表示状态：
```java
// 糟糕的写法
public static final int STATUS_PENDING = 0;
public static final int STATUS_PAID = 1;
// 问题1：可以传入任意 int 值，编译期不检查
// 问题2：0 和 1 没有自描述性，需要查文档
// 问题3：添加新状态时，所有 switch-case 都得改
```

### 解决方案：枚举替代常量
```java
enum OrderStatus {
    PENDING(0, "待支付"),
    PAID(1, "已支付"),
    SHIPPED(2, "已发货"),
    DELIVERED(3, "已签收"),
    CANCELLED(4, "已取消");

    private final int code;
    private final String desc;
    // 构造方法、getter、fromCode()...
}
```

### 优势
- **类型安全**：参数是 `OrderStatus` 类型，不会误传其他 int
- **可读性好**：`OrderStatus.PAID` 比 `1` 语义清晰
- **约束范围**：只允许预定义的几个值
- **自带方法**：`values()` 遍历、`valueOf()` 字符串转换

## 场景2：策略模式（替换大量 if/else）

### 问题
```java
// 臃肿的 if/else
if (type == 1) {
    price = price * 0.9;
} else if (type == 2) {
    price = price - 20;
} else if (type == 3) {
    price = price * 0.5;
}
// 添加新策略：改代码、改 if/else、可能忘记改某处
```

### 解决方案：枚举策略模式
```java
enum DiscountStrategy {
    VIP {
        @Override
        public double calculate(double price) {
            return price * 0.9;
        }
    },
    FULL_REDUCTION {
        @Override
        public double calculate(double price) {
            return price >= 100 ? price - 20 : price;
        }
    },
    HOLIDAY {
        @Override
        public double calculate(double price) {
            return price * 0.5;
        }
    };

    public abstract double calculate(double price);
}

// 使用——一行替代所有 if/else
double finalPrice = DiscountStrategy.VIP.calculate(200);
```

### 优势
- **开闭原则**：添加新策略只需增加一个枚举值，不改动已有代码
- **策略代码集中**：每个策略的计算逻辑写在各自的枚举常量中
- **编译期保证**：所有枚举值必须实现抽象方法，不遗漏

## 场景3：统一返回码（后端接口必备）

### 问题
```java
// 散落各处的返回码
return "{\"code\": 200, \"msg\": \"成功\"}";
return "{\"code\": 404, \"msg\": \"不存在\"}";
// code 和 msg 随意写，不一致，不统一
```

### 解决方案
```java
enum ResultCode {
    SUCCESS(200, "成功"),
    PARAM_ERROR(400, "参数错误"),
    UNAUTHORIZED(401, "未授权"),
    NOT_FOUND(404, "资源不存在"),
    SERVER_ERROR(500, "服务器内部错误");

    private final int code;
    private final String message;

    public String toResult(Object data) {
        return String.format("{\"code\": %d, \"message\": \"%s\", \"data\": \"%s\"}",
                code, message, data);
    }
}

// 使用——所有接口返回格式统一
return ResultCode.SUCCESS.toResult(user);
return ResultCode.PARAM_ERROR.toResult("用户名不能为空");
```

### 优势
- **全项目统一**：所有接口返回格式一致
- **前端友好**：前端可以按 code 码统一处理
- **可维护**：新增返回码只需加一个枚举值

## 枚举 vs 普通类的对比

| 特性 | 枚举 | 普通类 + static final |
|------|------|----------------------|
| 实例个数 | 有限、固定 | 无限 |
| 类型安全 | 编译期检查 | 依赖开发者约束 |
| switch 支持 | 原生支持 | int/String 支持 |
| 序列化 | 内置，单例保障 | 需额外处理 |
| 线程安全 | 天然线程安全 | 需自行保证 |
| 可读性 | 优秀 | 一般 |
