# 文本文件复制——字符缓冲流

## 一、为什么字符缓冲流最常用？

在 Java 中处理**文本文件**读取和写入时，`BufferedReader` + `BufferedWriter` 是事实上的标准选择：

| 特性 | 普通字符流 | 字符缓冲流 |
|------|-----------|-----------|
| 缓冲区 | 无 | 默认 8KB |
| 读取方式 | 逐字符 | 逐行 readLine() |
| IO 次数 | 每次 read 都访问磁盘 | 批量读取，大幅减少 IO |
| 性能 | 慢 | 快（通常数倍甚至数十倍提升） |

## 二、核心类

### BufferedReader
```java
// 构造：包装一个 Reader
BufferedReader reader = new BufferedReader(new FileReader("file.txt"));

// 核心方法
String line = reader.readLine(); // 读取一行，返回 null 表示结束
```

### BufferedWriter
```java
// 构造：包装一个 Writer
BufferedWriter writer = new BufferedWriter(new FileWriter("file.txt"));

// 核心方法
writer.write(str);     // 写入字符串
writer.newLine();      // 写入换行符（跨平台）
```

## 三、标准复制模板（最常用）

```java
try (BufferedReader reader = new BufferedReader(new FileReader("源.txt"));
     BufferedWriter writer = new BufferedWriter(new FileWriter("目标.txt"))) {

    String line;
    while ((line = reader.readLine()) != null) {
        writer.write(line);
        writer.newLine();
    }
} catch (IOException e) {
    e.printStackTrace();
}
```

**注意**：
- `readLine()` 不会返回行尾的换行符，所以写入时需要调用 `newLine()` 手动补上
- 使用 `try-with-resources` 自动关闭流，无需手动 close()

## 四、为什么不用 FileReader/FileWriter 直接复制？

```java
// 不推荐：每次 read() 都触发一次磁盘 IO，极其低效
FileReader fr = new FileReader("source.txt");
int ch;
while ((ch = fr.read()) != -1) {
    // 逐字符读写，磁盘 IO 次数 = 文件字符数
}
```

`BufferedReader` 一次从磁盘读取 **8KB（默认缓冲区大小）** 到内存，后续 readLine() 直接从内存读取，大幅减少磁盘寻道次数。

## 五、运行验证

```bash
javac TextFileCopy.java
java TextFileCopy
```

程序会：
1. 自动生成包含 1000 行文本的测试源文件 `source.txt`
2. 使用字符缓冲流将其复制到 `target.txt`
3. 对比无缓冲 vs 有缓冲的性能差异

# 补充：任意文件复制——字节缓冲流万能复制

## 一、字符流 vs 字节流

| | 字符流 | 字节流 |
|------|------|------|
| 基类 | Reader/Writer | InputStream/OutputStream |
| 单位 | 字符(char) | 字节(byte) |
| 适用 | 纯文本文件(.txt, .java, .json...) | **所有文件类型** |
| 缓冲流 | BufferedReader/BufferedWriter | BufferedInputStream/BufferedOutputStream |

**字符流只能处理文本**——它会把字节按编码表转成字符。对于图片、视频等二进制文件，解码过程会导致数据损坏。

**字节流可以处理所有文件**——它原样搬运字节，不做任何转换，所以是"万能"的。

## 二、字节缓冲流标准模板

```java
try (BufferedInputStream bis = new BufferedInputStream(new FileInputStream("源文件"));
     BufferedOutputStream bos = new BufferedOutputStream(new FileOutputStream("目标文件"))) {

    byte[] buffer = new byte[8192]; // 8KB 缓冲区
    int len;
    while ((len = bis.read(buffer)) != -1) {
        bos.write(buffer, 0, len);
    }
} catch (IOException e) {
    e.printStackTrace();
}
```

这个模板适用于**任何文件**：txt、png、mp3、mp4、zip、class...

## 三、核心要点

- **8KB 缓冲区**：一次读 8192 字节，平衡了内存和 IO 次数
- **read(buffer)**：返回实际读取的字节数，读到末尾返回 -1
- **write(buffer, 0, len)**：只写入有效数据，避免写入上次残留
- **万能**：字节流不关心文件内容是什么，直接搬运原始字节
